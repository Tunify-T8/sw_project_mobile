## 2. Required API Endpoints

The backend must implement these endpoints to manage device tokens:

### 2.1 Register Device Token

**Endpoint:** `POST /api/notifications/device-tokens`

**Authentication:** Required (Bearer Token in Authorization header)

**Request Body:**
```json
{
  "token": "FCM_TOKEN_STRING",
  "fcmToken": "FCM_TOKEN_STRING",
  "platform": "android" | "ios"
}
```

**Example cURL:**
```bash
curl -X POST https://tunify.duckdns.org/api/notifications/device-tokens \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "token": "eR5Jvui6RfCxjKvUg...",
    "fcmToken": "eR5Jvui6RfCxjKvUg...",
    "platform": "android"
  }'
```

**Success Response (200 OK):**
```json
{
  "success": true,
  "message": "Device token registered successfully"
}
```

**Error Response (400/401):**
```json
{
  "success": false,
  "error": "Invalid or missing token"
}
```

**Backend Implementation Notes:**
- Extract JWT from Authorization header
- Decode JWT to get `userId`
- Store token in database with fields:
  - `userId` (from JWT)
  - `fcmToken` (the token string)
  - `platform` (`android` or `ios`)
  - `registeredAt` (timestamp)
  - `isActive` (boolean, default true)
- If token already exists for this user+platform, update it (don't create duplicate)
- No need for additional validation beyond checking token format

---

### 2.2 Unregister Device Token (Logout)

**Endpoint:** `DELETE /api/notifications/device-tokens`

**Authentication:** Required (Bearer Token in Authorization header)

**Request Body:**
```json
{
  "token": "FCM_TOKEN_STRING",
  "fcmToken": "FCM_TOKEN_STRING",
  "platform": "android" | "ios"
}
```

**Success Response (200 OK):**
```json
{
  "success": true,
  "message": "Device token unregistered successfully"
}
```

**Backend Implementation Notes:**
- Extract JWT to get `userId`
- Find and delete the token record for that user
- If token doesn't exist, still return 200 (idempotent)
- This is called on logout/sign out all

## 3. Sending Push Notifications

### 3.1 Database Schema Example

Store registered tokens in a table/collection:

```sql
-- SQL Example (PostgreSQL/MySQL)
CREATE TABLE device_tokens (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  fcm_token TEXT NOT NULL,
  platform VARCHAR(10) NOT NULL CHECK (platform IN ('android', 'ios')),
  is_active BOOLEAN DEFAULT true,
  registered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_used_at TIMESTAMP,
  UNIQUE(user_id, fcm_token)
);

-- Indexes for fast lookup
CREATE INDEX idx_device_tokens_user_id ON device_tokens(user_id);
CREATE INDEX idx_device_tokens_active ON device_tokens(is_active) WHERE is_active = true;
```

```javascript
// MongoDB Example
db.deviceTokens.createIndexes([
  { key: { userId: 1, fcmToken: 1 }, unique: true },
  { key: { isActive: 1 } },
  { key: { registeredAt: 1 } }
]);
```

### 3.2 Sending Notifications via Firebase Admin SDK

#### Node.js Example

```javascript
const admin = require('firebase-admin');

// Initialize Firebase Admin SDK (use service account key)
admin.initializeApp({
  credential: admin.credential.cert(serviceAccountKey),
  projectId: 'tunify-3d931'
});

/**
 * Send notification to a single user's device
 * @param {string} userId - User ID
 * @param {Object} notification - { title, body, data }
 */
async function sendNotificationToUser(userId, notification) {
  try {
    // Get all active tokens for this user
    const tokens = await db.collection('deviceTokens')
      .where('userId', '==', userId)
      .where('isActive', '==', true)
      .get();

    if (tokens.empty) {
      console.log(`No tokens found for user ${userId}`);
      return;
    }

    const failedTokens = [];
    const successCount = 0;

    // Send to each token
    for (const doc of tokens.docs) {
      const { fcmToken, platform } = doc.data();
      
      try {
        const message = {
          notification: {
            title: notification.title,
            body: notification.body
          },
          data: notification.data || {},
          token: fcmToken
        };

        // Platform-specific options
        if (platform === 'android') {
          message.android = {
            priority: 'high',
            notification: {
              channelId: 'tunify_notifications',
              clickAction: 'FLUTTER_NOTIFICATION_CLICK'
            }
          };
        } else if (platform === 'ios') {
          message.apns = {
            headers: {
              'apns-priority': '10'
            }
          };
        }

        await admin.messaging().send(message);
        successCount++;
      } catch (error) {
        console.error(`Failed to send to token ${fcmToken}:`, error);
        
        // Mark token as invalid if unregistered
        if (error.code === 'messaging/invalid-registration-token' ||
            error.code === 'messaging/registration-token-not-registered') {
          failedTokens.push(doc.id);
        }
      }
    }

    // Clean up invalid tokens
    for (const tokenId of failedTokens) {
      await db.collection('deviceTokens').doc(tokenId).update({
        isActive: false
      });
    }

    console.log(`Notification sent: ${successCount} success, ${failedTokens.length} failed`);
  } catch (error) {
    console.error('Error sending notification:', error);
  }
}

/**
 * Send notification to multiple users
 */
async function sendNotificationToUsers(userIds, notification) {
  for (const userId of userIds) {
    await sendNotificationToUser(userId, notification);
  }
}

// Example: Send when a new follower follows someone
async function handleNewFollower(followerId, followedUserId) {
  await sendNotificationToUser(followedUserId, {
    title: 'New Follower',
    body: `Someone started following you`,
    data: {
      type: 'follower',
      followerId: followerId,
      action: 'open_profile'
    }
  });
}

// Example: Send when a track gets a like
async function handleTrackLiked(trackOwnerId, likerUsername) {
  await sendNotificationToUser(trackOwnerId, {
    title: 'Track Liked',
    body: `${likerUsername} liked your track`,
    data: {
      type: 'like',
      trackId: trackId,
      action: 'open_track'
    }
  });
}
```

#### Python Example (if using Django/FastAPI)

```python
from firebase_admin import credentials, initialize_app, messaging
from datetime import datetime

# Initialize Firebase
cred = credentials.Certificate('path/to/serviceAccountKey.json')
initialize_app(cred, {
    'projectId': 'tunify-3d931'
})

def send_notification_to_user(user_id: str, title: str, body: str, data: dict = None):
    """Send notification to all active devices of a user"""
    
    # Get tokens from database
    tokens = DeviceToken.objects.filter(
        user_id=user_id,
        is_active=True
    ).values_list('fcm_token', 'platform')
    
    if not tokens:
        print(f"No tokens found for user {user_id}")
        return
    
    failed_tokens = []
    success_count = 0
    
    for fcm_token, platform in tokens:
        try:
            message = messaging.Message(
                notification=messaging.Notification(
                    title=title,
                    body=body
                ),
                data=data or {},
                token=fcm_token
            )
            
            # Send message
            response = messaging.send(message)
            success_count += 1
        except Exception as e:
            print(f"Failed to send to {fcm_token}: {str(e)}")
            if "invalid-registration-token" in str(e):
                failed_tokens.append(fcm_token)
    
    # Mark invalid tokens as inactive
    DeviceToken.objects.filter(
        fcm_token__in=failed_tokens
    ).update(is_active=False)
    
    print(f"Sent to {success_count}, failed: {len(failed_tokens)}")
```

## 4. Notification Payload Structure

### 4.1 Payload Types

**Type 1: Display Notification** (shown in notification tray)
```json
{
  "notification": {
    "title": "Tunify",
    "body": "You have a new notification"
  },
  "data": {
    "type": "like",
    "trackId": "track123",
    "action": "open_track"
  }
}
```

**Type 2: Data Notification** (silent, data-only, requires app handling)
```json
{
  "data": {
    "type": "message",
    "messageId": "msg456",
    "from": "user789"
  }
}
```

**Type 3: Combined** (both notification and data)
```json
{
  "notification": {
    "title": "New Message",
    "body": "You have 1 new message"
  },
  "data": {
    "type": "message",
    "conversationId": "conv123"
  }
}
```

### 4.2 Data Field Guidelines

Supported `type` values (app will handle these):
- `like` - track/playlist liked
- `follower` - new follower
- `message` - new message/DM
- `comment` - new comment on track
- `repost` - track reposted
- `mention` - user mentioned
- `promotion` - promotional message

**Required data fields:**
- `type` - notification type
- `action` - what to do when tapped (e.g., `open_track`, `open_profile`, `open_message`)

**Optional data fields:**
- `id` - entity ID (trackId, userId, messageId, etc.)
- `username` - related username
- `payload` - additional JSON data

## 7. Common Issues & Solutions

### Issue: Tokens not syncing to backend
**Solutions:**
- Verify backend endpoint path is exactly `/api/notifications/device-tokens`
- Check JWT token is valid in Authorization header
- Verify backend returns 200 status code
- Check backend logs for auth/parsing errors

### Issue: Notifications not arriving
**Solutions:**
- Verify FCM token is correct (check in app logs)
- Verify same Firebase project is used (project ID in service account)
- Check Firebase service account key has messaging permissions
- Verify `channelId` matches Android manifest (`tunify_notifications`)
- Check app permissions: `POST_NOTIFICATIONS` on Android 13+

### Issue: Invalid token error
**Solutions:**
- Tokens expire, regenerate on app restart
- Mark expired tokens as inactive in DB
- Remove tokens older than 30 days periodically
